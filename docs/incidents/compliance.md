# Compliance incident and overdue request runbook

1. Identify the overdue request or suspected personal-data incident without
   copying sensitive payloads into chat, tickets or logs. Appoint an owner and
   preserve audit evidence.
2. Confirm identity, scope, jurisdiction and applicable deadline. Mark a request
   `IN_PROGRESS`, notify legal/support and record expected completion. Never mark
   it complete solely to clear an alert.
3. For suspected disclosure, contain access, rotate credentials and determine
   data categories, subjects, jurisdictions, time window and consequences.
4. Counsel decides regulator and user notification duties. Record the decision,
   approver and facts; engineering must not infer non-reportability.
5. Validate exports, corrections, restrictions or deletions with a second
   reviewer, update the audit trail and verify downstream processor action.
